# Product Context — JMR Visuals Portfolio

## Register

**Surface:** Brand
**Type:** Creative portfolio — design student specializing in print projects and 3D design

## Audience

- Art directors and creative leads at studios, agencies, and publishing houses
- Fellow designers and 3D artists in the ArtStation/Behance community
- University faculty and fellowship reviewers assessing student work
- Potential freelance clients seeking print design or 3D visualization

## Voice

| Trait        | Description                                                                 |
| ------------ | --------------------------------------------------------------------------- |
| Tone         | Quiet confidence — the work speaks; the site amplifies without shouting     |
| Register     | Art-world editorial, not tech-startup conversational                        |
| Copy length  | Minimal. Captions, not paragraphs. Let images breathe.                     |
| Personality  | Curated gallery, not a blog. Think monograph, not LinkedIn.                |

## Brand Pillars

1. **Materiality** — The site should feel tactile: paper textures, ink weight, physicality of print
2. **Dimension** — 3D work demands spatial awareness; the layout should hint at depth
3. **Precision** — Clean typographic craft reflecting the discipline of print design
4. **Asymmetry** — Deliberate tension and imbalance over grid conformity

## Anti-References (What This Must NOT Look Like)

- Generic rounded-card portfolio grids (Dribbble template aesthetic)
- Neon gradients, glassmorphism, or "SaaS hero" sections
- Inter/system-font default stacks
- Purple-to-blue color ramps
- Cards nested inside cards
- Centered-everything symmetry with equal-width columns
- Cookie-cutter project thumbnails in identical aspect ratios
- Stock photo hero banners
- Hamburger-menu-only navigation on desktop

## Content Source

Primary content and assets: https://www.artstation.com/jmr_visuals
Content categories: Print projects, 3D designs/mockups, potential motion/turntable renders

## Deployment

Static site (HTML/CSS/JS) → GitHub Pages or Netlify
No server-side rendering at runtime. SSG build only.
